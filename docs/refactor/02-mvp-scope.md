# MVP Scope

日期：2026-04-24

## 第一阶段目标

第一阶段只解决一个问题：让用户能稳定完成一次高质量自动配音，并在失败时知道怎么修。

主链路：

```text
上传视频
  -> 生成/上传 source.srt
  -> 智能分句
  -> 可选短句合并
  -> 翻译
  -> 一句一句配音
  -> Review 问题句
  -> 重配改动句
  -> 导出结果
```

用户卡住时的辅助链路：

```text
用户不知道下一步 / 遇到报错 / 不理解参数
  -> 打开右下角 Agent
  -> 描述问题或粘贴错误
  -> Agent 给出具体下一步建议
  -> 用户手动执行建议
```

## 必须保留的能力

| 能力 | 要求 |
|---|---|
| 上传媒体 | 支持当前已有视频/音频上传方式。 |
| 上传 SRT | 支持 source 字幕和 translated 字幕两种模式。 |
| ASR | 没有上传字幕时继续自动识别。 |
| 智能分句 | source.srt 最终保存分句后的结果。 |
| 可选短句合并 | 默认关闭，用户开启后按时间目标合并。 |
| 翻译 | 继续使用 DeepSeek/OpenAI-compatible 接口。 |
| Auto Dubbing V2 | 作为主推荐路径保留。 |
| Review | 能查看问题句并修改译文。 |
| Save and Redub | 只重配改动句，并保留原始配置。 |
| 导出 | 输出视频、音频、source/translated/bilingual/final subtitles。 |
| 历史 batch | 能加载、review、redub。 |
| 使用帮助 | Agent V1 能解释当前页面常见操作和错误。 |

## 关键用户场景

| 场景 | 成功标准 |
|---|---|
| 只上传视频 | 系统自动 ASR、分句、翻译、配音并产出可 review 的结果。 |
| 上传 source 字幕 | 跳过 ASR，但仍执行智能分句、可选短句合并、翻译、配音。 |
| 上传 translated 字幕 | 跳过 ASR 和翻译，直接使用已翻译字幕配音。 |
| 只想测试片段 | 用户选择时间区间后，产物和字幕时间轴保持一致。 |
| 配音有问题句 | Review 面板能定位问题句，用户改译文后只重配改动句。 |
| 任务失败 | UI 或 Agent 能说明失败类型和下一步检查项。 |

## 字幕规则

| 项目 | 决策 |
|---|---|
| 第一步分句 | 每句末尾必须优先是句号，太长才按逗号拆。 |
| 第二步合并 | 只合并，不拆分凑时长。 |
| 合并开关 | 默认关闭。 |
| 合并依据 | 按时间目标，不按字数。 |
| tolerance | `round(target_seconds / 3)`，至少 1 秒。 |
| 上传字幕 | 上传 source.srt 和 ASR 生成 source.srt 都走同一套规则。 |

## 配音规则

| 项目 | 决策 |
|---|---|
| 合成粒度 | 一句一句配音。 |
| 音色 | 固定使用统一 voice reference。 |
| 情绪 | 每句使用对应原句音频作为 emotion reference。 |
| 时间 | 尽量接近原句时间，不追求绝对强行压缩。 |
| 多人识别 | 第一阶段不做 diarization / pyannote。 |
| 主 backend | Index-TTS。 |
| 备胎 backend | OmniVoice，后续通过独立 adapter 接入。 |

## Agent V1 Scope

| 项目 | 范围 |
|---|---|
| 入口 | 右下角浮动抽屉。 |
| 能力 | 使用帮助、报错解释、下一步建议。 |
| 模型 | DeepSeek `deepseek-chat`。 |
| 上下文 | 用户问题 + 当前页面名。 |
| 不做 | 不读本地文件、不读 manifest、不执行操作。 |

Agent V1 必须覆盖的常见问题：

| 问题 | 回答方向 |
|---|---|
| 不知道下一步 | 根据当前页面解释下一步操作。 |
| DeepSeek key 报错 | 解释 key 来源、输入位置、环境变量。 |
| Index-TTS 不可用 | 提醒检查本地服务和 health endpoint。 |
| 字幕太零散 | 解释智能分句和短句合并开关。 |
| 不知道 source/translated 区别 | 解释 source 需要翻译，translated 会跳过翻译。 |
| redub 失败 | 建议保留现有产物，查看错误摘要后重试。 |

## 明确不做

| 不做 | 原因 |
|---|---|
| Google Login | 本地工具阶段不是核心瓶颈。 |
| 收费系统 | 核心价值还在打磨。 |
| 多租户 | 会显著增加复杂度。 |
| 团队协作 | 当前主要是个人工作流。 |
| 云端任务队列 | 先规范本地任务状态。 |
| 自动说话人识别 | 已确认主链路不需要。 |
| React/Vue 重写 | 会放大重构风险。 |
| 彻底删除旧 CLI | 旧入口先保留兼容。 |

## 第一阶段验收

| 验收项 | 标准 |
|---|---|
| 主链路 | 1 分钟视频能完成上传、字幕、翻译、配音、review、导出。 |
| 字幕 | source.srt 应用智能分句和可选短句合并。 |
| 配音 | 逐句合成，不依赖自动说话人识别。 |
| 失败 | DeepSeek、Index-TTS、ASR、ffmpeg 类失败都有可读提示。 |
| Agent | 能解释常见使用问题，不执行危险操作。 |
| 回归 | 现有测试和新增 contract tests 通过。 |

## Review 1 结论

产品原则和 MVP 范围确认后，第一阶段只优化“完成一次自动配音”的闭环体验。任何账号、收费、多租户、自动说话人识别、前端框架迁移，都不应在第一阶段插入。
